# shotframe — basic example

A minimal, runnable shotframe config that exercises every path:

- **Path A** (real screenshots) for App Store, Google Play and Chrome Web Store
  targets, pointing `source:` at images in `./shots`.
- **Path B** (synthetic preset) — the `iphone69-hero` target draws a fully
  synthetic screen via [`presets/hero.ts`](./presets/hero.ts), with no screenshot.

The sample screenshots are generated by a tiny zero-dependency PNG encoder
(`gen-shots.mjs`) so the example is self-contained.

## Run it

From this folder (`examples/basic`):

```bash
# 1) generate the sample source screenshots into ./shots
node gen-shots.mjs          # or: pnpm shots

# 2) headless batch render → ./out/<store>/<id>.<ext>
node ../../packages/cli/dist/index.js render --config ./shotframe.config.ts --out ./out
#   (installed globally you'd just run: shotframe render)

# 3) live browser editor (Ctrl+C to stop)
node ../../packages/cli/dist/index.js studio --config ./shotframe.config.ts
```

`render` requires Chromium once: `npx playwright install chromium`.

## What you get

```
out/
  appstore/iphone69-1.jpg      1290×2796  (Path A, real screenshot)
  appstore/iphone69-hero.jpg   1290×2796  (Path B, synthetic preset)
  play/androidp-1.jpg          1080×1920  (Path A)
  chrome/screenshot-1.png      1280×800   (Path A, browser chrome)
```

Add `--store appstore` (or `play`/`chrome`) to render just one store.

> Note: this example uses the workspace packages directly (`workspace:*`). In a
> real project you'd `npm i -D @shotframe/cli` and run `npx shotframe render`.
